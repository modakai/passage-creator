# 后续功能任务

以下能力在 `docs/ui/preview.html` 中有展示，但当前独立用户端先保留可实现的主流程，未强行伪造后台能力。

## 需要后端或产品能力补齐

- 首页「本月创作效率」暂为静态展示。后续需要新增用户端统计接口，返回节省时间、完成作品数、等待确认数和生成配图数。
- 额度页暂未接入真实余额接口。后续应复用或新增用户端积分汇总接口，展示余额、充值、消耗和模型用量明细。
- 会员方案和充值按钮暂为展示态。当前系统有积分计费基础，但缺少用户端在线支付、套餐购买和订单状态确认闭环。
- 任务页目前通过文章和小红书两个列表接口拼装，后续可补充统一任务聚合接口减少前端合并逻辑。
- 个人中心偏好暂为展示态。后续需要新增默认模型、写作风格、文章长度、通知开关等用户偏好 API。
- 生成图片下载按钮在文章和小红书流程中可继续增强为后端代理下载，避免 OSS 或第三方图片 CORS 限制。

## 可暂时忽略

- 原型中的复杂导航切换动画不影响核心创作流程，可在主流程稳定后再做动效增强。
- 原型中的模拟作品缩略图可先使用渐变占位，避免引入无来源图片资源。
- 原型中的团队套餐、品牌风格库等团队能力当前不属于个人用户端核心流程。
